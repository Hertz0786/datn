1. Mongoose Deprecation Warnings (LOW PRIORITY)
------------------------------------------------
Location: be/src/models/*.js

Problem: All `findOneAndUpdate()` and `findOneAndReplace()` calls pass
`new: true` which is deprecated in Mongoose 6+. Mongoose recommends
`returnDocument: 'after'` instead.

Impact: Warning messages in server logs only. No functional impact.

Fix: Replace every occurrence of:
  Model.findOneAndUpdate(query, update, { new: true })
with:
  Model.findOneAndUpdate(query, update, { returnDocument: 'after' })

2. WebSocket Authentication (MEDIUM PRIORITY)
----------------------------------------------
Problem: The Socket.IO handshake does not currently verify JWT tokens
on connection. Any client can connect to the WebSocket and listen for
events on any room (post:join, chat join, etc.).

Risk: Users can potentially listen to messages in chat rooms they are
not members of, or receive real-time updates for posts they should not
see.

Fix Options:
  a) Verify JWT in Socket.IO handshake middleware:
       const { token } = socket.handshake.auth;
       try { socket.user = jwt.verify(token, secret); } catch { socket.disconnect(); }

  b) Verify user membership on each room join:
       - For post rooms: check user can see the post before joining.
       - For chat rooms: check user is a chat member before joining.

Recommendation: Implement option (a) as the primary gate, plus (b) as
defense-in-depth for sensitive rooms.

3. Peer Disconnect Retry Logic (MEDIUM PRIORITY)
-----------------------------------------------
Location: be/src/services/calls.js, fe/lib/features/call/call_screen.dart

Problem: When a WebRTC peer disconnects abruptly (e.g. network drop),
the current implementation may not automatically retry the connection.

Current behavior: The call shows "Reconnecting..." or hangs.

Enhancement: Implement exponential backoff retry:
  1. On disconnect event, wait 1s, 2s, 4s, 8s (max 3 attempts).
  2. Re-create RTCPeerConnection and re-exchange SDP offers.
  3. After max retries, show "Connection lost" with manual retry button.
  4. Send 'call:peer_disconnected' event so the other peer knows.

Also add ICE restart: when the connection fails due to network change,
call pc.restartIce() before creating a new offer.

---------------------------------------------
Location: be/src/jobs/ (new directory, or scheduled task)

Problem: Stale call rooms and chat sessions may accumulate in Redis/Memory
if participants crash or lose connectivity without sending a proper end.

Enhancement: Add a periodic cleanup job (runs every 5 minutes):
  - Find call rooms older than 30 minutes with status IN_PROGRESS.
  - For each stale room, call cleanupCallRoom() to:
      - Set status to ENDED.
      - Calculate duration.
      - Save to history.
      - Remove from Redis.
